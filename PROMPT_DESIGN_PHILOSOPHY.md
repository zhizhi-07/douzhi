# 🎯 提示词设计哲学 - 从模板到原则

## 🔍 问题发现

用户指出了一个致命问题：

> "你这样弄AI就会学，翻来覆去就这么几个样子，你是去指导让他学习，不是给他模板"

### ❌ 之前的做法（错误）

```
给3个完整场景示例：

**场景示例1：用户发'今天好累'**
- 0秒：女友A看到，立刻评论'宝贝辛苦了❤️'
- 5秒：好友C看到，私聊关心'看你发朋友圈了，最近工作压力大吗？'
- 8秒：暧昧对象B看到，评论'要不要我给你按摩？😊'
- 12秒：女友A看到B的评论，立刻回复B'你谁啊？😅'
...
```

**问题**：
1. AI会机械模仿这些台词
2. 每次都是"宝贝辛苦了"、"又发什么神经"
3. 翻来覆去就这几个样子
4. 缺乏创造力和多样性

## 💡 学习MoM果实预设

分析了桌面上的`【MoM】果实4.75丨喵喵电波丨0819.json`，发现优秀预设的核心原则：

### ✅ 正确做法

#### 1. 给原则，不给示例
```
- 角色对<user>的好感度上涨非常缓慢
- 表白或者喜欢必须经过重要事件
- 角色会因为<user>的不足而产生偏见和嫌弃
```

#### 2. 用正反案例对比
```
✅ 正面案例：
- 窗帘是拉上的，房间里只开了一盏桌角的台灯

❌ 负面案例（禁止）：
- 房间里昏暗而压抑
```

#### 3. 强调禁忌和要求
```
写作准则：
- 严禁使用一切抽象、象征、意象性语言
- 必须用明确具体的行为、对话或内心独白表达情节
```

#### 4. 核心是引导思维方式，而非提供模板

## 🎯 新的提示词设计

### 1. 编排原则（代替具体示例）

```markdown
### 1. 角色互动强度的真实性
- 关系越亲密，互动越主动（评论/私聊/回复）
- 关系越陌生，行为越保守（沉默/仅点赞）
- 好感度上涨**非常缓慢**

### 2. 评论内容的合理性
- 亲密关系：可以用亲密称呼
- 普通关系：礼貌关心
- 陌生关系：简单互动或沉默
```

### 2. 正反案例（简洁明了）

```markdown
**✅ 正确：基于可见信息的反应**
- 看到陌生人评论"要不要我给你按摩？" → 质疑"你谁啊？🙄"
- 看到别人评论"老婆辛苦了" → 意外"哈？😅"或选择沉默

**❌ 错误：表现出上帝视角**
- ✗ 直接说"你这个小三别勾引我男朋友"（怎么知道对方想勾引？）
- ✗ 说"你们俩是不是在一起了"（怎么知道别人的关系？）
```

### 3. 编排禁忌（防止套路化）

```markdown
## 编排禁忌（严禁）

### 🚫 禁止模板化
- 不要每次都让女友说"宝贝辛苦了"
- 不要每次都让损友说"又发什么神经"
- **每个角色的反应应该基于朋友圈内容和当下情境**

### 🚫 禁止套路化冲突
- 不要每次都让角色在评论区撕X
- 冲突要自然，不是为了冲突而冲突
- **沉默、冷淡、礼貌也是一种真实反应**
```

### 4. 核心原则（强调思考）

```markdown
### 1. 真实优先于戏剧
- 角色的反应必须符合其性格、关系、认知范围
- 不为了制造冲突而强行编排

### 2. 每次编排都要创新
- 根据朋友圈内容定制反应
- 不要套用固定模板
- 同样的角色在不同情境下会有不同反应

### 5. 角色视角是底线
- 绝对不能暴露上帝视角
- 这是最重要的原则
```

## 📊 对比总结

| 维度 | 之前（模板式） | 现在（原则式） |
|------|--------------|--------------|
| **示例** | 3个完整场景 | 简短正反案例 |
| **台词** | 给具体对话 | 给原则和方向 |
| **限制** | 暗示性的 | 明确的禁忌 |
| **目标** | 让AI模仿 | 让AI思考 |
| **结果** | 机械重复 | 灵活创新 |

## 🎓 学到的经验

### 1. 提示词不是示例库
- 不要试图通过示例让AI学会
- 示例会变成AI唯一知道的模板

### 2. 提示词是原则和约束
- 告诉AI"什么是对的"
- 更重要：告诉AI"什么是错的"

### 3. 正反案例要简洁
- 不要长篇大论解释
- 直接对比：✅ 这样 vs ❌ 那样

### 4. 禁忌比建议更重要
- "不要做什么"比"要做什么"更清晰
- 明确的禁止能避免套路化

### 5. 相信AI的创造力
- AI不需要手把手教
- 给清晰的原则，AI会自己创造
